Example 3: Analyzing a Work in Progress

In this example, we will analyze a recipe that we are working on for a middle school home economics class. Our target age group will be sixth-seventh grade reading and we will use Readability Studio to help us achieve this. The first step is to open the New Project wizard and select the book. Select New Project... from the File menu to open the New Project wizard, select the option Read text from a file, and click the Next button.

Next, you will see the Select File dialog. You can either type in the file path of the document or browse for it. Click the button next to the file path text box to browse for it:

This will open the Select Document to Analyze dialog. Choose the file that you want to analyze and click the OK button to accept it:

You will then return to the Select File dialog. Click the Next button to proceed to the Readability Tests dialog, as shown below:

Because this is a short, terse document containing bulleted lists, we will consider this document to be a form. Select the Recommend tests based on document type and click the Next button to proceed to the Select Document Type dialog, as shown below:

Select Form (application, résumé, brochure, menu, academic test) and click the Next button. The next dialog will be the Select Highlighted Documents dialog, as shown below:

Select all of the checkboxes so that we will have all of the highlighted text views available. Click the Next button to proceed to the Select Difficult Words Lists dialog, as shown below:

Select all of the checkboxes so that we will have all of the difficult word lists available and click the Finish button to begin analyzing the project.

At this point we can review the test scores of the recipe (by clicking on the Readability Scores icon in the left window) to help determine which reading age it is appropriate for. As we can see below, it appears to be readable for at least readers in the seventh to eighth grade range or above. This is higher than our target audience and we may need to make some improvements to the recipe to make it easier to read.

To help understand why this recipe may be difficult for younger readers, click on the Difficult Words icon on the project sidebar to display various lists of difficult words. Select the 6+ character words tab and click on the Character count column header twice to sort the words in descending order (where the longest words appear at the top of the list). As illustrated below, there are many words containing ten or more characters, which can be very difficult for most readers. Also, when looking at the 3+ syllable words tab we can see that many of these same words also have high syllable counts and are rather difficult and uncommon words.

To view these words in their original context, click on the View Document icon on the project sidebar and then click on the 6+ character words tab, as illustrated below:

You may also click on any of the other tabs to view other difficult words in their original context, such as 3+ syllable words or unfamiliar Dale-Chall words.

To begin simplifying this recipe we will open the original document in Microsoft® Wordpad and begin replacing some of these difficult words. Words such as "one-hundred-and-sixty-five" could be shortened to "165", and difficult words such as "disseminate", "incorporate", "epidermis", "abdominal ", and "transparent" could also be replaced with shorter, simpler synonyms. As illustrated below, we have replaced a few words, such as substituting "chest" for "abdominal", "skin" for "epidermis", "use" for "utilize", etc.

The next item to review is the overly long sentences. Click on the View Document icon on the project sidebar and then click on the Long Sentences tab, as illustrated below:

As we scroll through this window, we can see a few sentences that are highlighted because they are overly long. Now we will return to the original document and first remove some of the wordiness from the first sentence:

Another sentence is a run-on sentence and is better off being split into two separate sentences, as shown below:

After we are finished reworking the original document, we will want to analyze it again to see if it now meets our target audience. Rather than starting all over again my making a new project, we will link this project directly to the recipe document. By default, Readability Studio saves the document's original text in the project; however, we can change this so that the project instead links to the source document. Whenever we open the project, it will reload and reanalyze the recipe document. To do this, select Project Settings... from the File menu to open the Properties dialog. Select the Document Analysis icon and then select the option Reload the source document when opening project. Click the OK button to accept.

Click the Reload button on the toolbar (as shown below), or select it from the Edit menu. Readability Studio will now reload the recipe document, meaning that all of the improvements that we had previously made will now be imported into the project. As you can see below, our grade level scores now fall within our target audience and this recipe will be far more readable for our home economics class.

Note that you can click the Reload button at anytime to reanalyze the source document as you edit it externally. Also, whenever you open this project it will automatically reload the source document (in this case the recipe file).
