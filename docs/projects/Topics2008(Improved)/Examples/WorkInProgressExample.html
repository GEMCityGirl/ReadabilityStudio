Example 3: Analyzing a Work in Progress

In this example, we will analyze a recipe that we are working on for a middle school home economics class. Our target age group will be sixth-seventh grade reading and we will use Readability Studio to help us achieve this. The first step is to open the New Project wizard and select the book. Select New Project... from the File menu to open the New Project wizard, select the option Read text from a file, and click the Next button.

Next, you will see the Select File dialog. You can either type in the file path of the document or browse for it. Click the button next to the file path text box to browse for it:

This will open the Select Document to Analyze dialog. Choose the file that you want to analyze and click the OK button to accept it:

You will then return to the Select File dialog. Click the Next button to proceed to the Select Document Structure dialog, as shown below:

On this dialog, we will specify how this document is formatted. The first option, Narrative text, is meant for documents that consists of the standard sentence and paragraph structure. The second option, Narrative text (fragmented), is the same as narrative text, except that some of the sentences may have unusual multiline formatting. The third option, Non-narrative text, is meant for documents that contain very few sentences and mostly consist of sentence fragments.

This document does have some headers and list items in it (which are sentence fragments). However, we are not interested in analyzing these fragments, and the document does contain a fair amount of narrative text that we are interested in. Because of this, the Non-narrative text is not a candidate.

This document contains narrative text, and this text does not have any unusual multiline formatting. Therefore, the Narrative text (fragmented) option is not a candidate.

This leaves the option Narrative text. This option applies to most documents, and this option does describe our recipe. Select Narrative text and click the Next button to proceed to the Readability Tests dialog, as shown below:

This file does not belong to any particular industry, but we do know that it is an instructional document. Select the Recommend tests based on document type and click the Next button to proceed to the Select Document Type dialog, as shown below:

Because this is an instructional document containing small narrative blocks, we will consider this to be a technical document. Select Technical document or form and click the Next button. The next dialog will be the Select Highlighted Documents dialog, as shown below:

Select all the checkboxes so that we will have all the highlighted text views available. Click the Next button to proceed to the Select Difficult Words Lists dialog, as shown below:

Select all the checkboxes so that we will have all the difficult word lists available and click the Finish button to begin analyzing the project.

At this point we can review the test scores of the recipe to determine which reading age it is appropriate for. Click on the Readability Scores icon in project sidebar to display the scores. As we can see below, it appears to be readable for at least readers in the seventh to eighth grade range or above. This is higher than our target audience and we may need to make some improvements to this recipe to make it easier to read.

To help understand why this recipe may be difficult for younger readers, click on the Difficult Words icon on the project sidebar. This will display various lists of difficult words. Select the 6+ character words tab and click on the Character count column header twice to sort the words in descending order. As illustrated below, there are many words containing ten or more characters, which can be very difficult for most readers. Also, when looking at the 3+ syllable words tab we can see that many of these same words also have high syllable counts and are rather difficult.

To view these words in their original context, click on the View Document icon on the project sidebar and then click on the 6+ character words tab, as illustrated below:

You can click on any of the other tabs to view other difficult words in their original context, such as 3+ syllable words or unfamiliar Dale-Chall words.

To begin simplifying this recipe we will open the original document in Microsoft® Wordpad and begin replacing some of these difficult words. Words such as "one-hundred-and-sixty-five" could be shortened to "165", and difficult words such as "disseminate", "incorporate", "epidermis", "abdominal ", and "transparent" could also be replaced with shorter, simpler synonyms. As illustrated below, we have replaced a few words, such as substituting "chest" for "abdominal", "skin" for "epidermis", "use" for "utilize", etc.

The next item to review is the overly long sentences. Click on the View Document icon on the project sidebar and then click on the Long Sentences tab, as illustrated below:

As we scroll through this window, we can see a few sentences that are highlighted because they are overly long. Now we will return to the original document and remove some wordiness from the first sentence:

Another sentence is a run-on sentence and is better off being split into two separate sentences, as shown below:

After we are finished reworking the original document, we will want to analyze it again to see if it now meets our target audience. Rather than starting all over again my making a new project, we will link this project directly to the recipe document. By default, Readability Studio saves the document's original text in the project; however, we can change this so that the project instead links to the source document. Whenever we open the project, it will reload and reanalyze the recipe document. To do this, select Project Properties... from the File menu to open the Properties dialog. Select the Document Analysis icon and then select the option Reload the source document when opening project. Click the OK button to accept.

Click the Reload button on the toolbar (as shown below), or select it from the Edit menu. Readability Studio will now reload the recipe document, meaning that all the improvements that we had previously made will now be imported into the project. As you can see below, our grade level scores now fall within our target audience and this recipe will be far more readable for our home economics class.

Note that you can click the Reload button at anytime to reanalyze the source document as you edit it externally. Also, whenever you open this project it will automatically reload the source document (in this case the recipe file).
