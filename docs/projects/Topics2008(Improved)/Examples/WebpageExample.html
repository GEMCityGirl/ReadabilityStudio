Example 4: Analyzing a Webpage

In this example, we will create a project that reads and analyzes a webpage. Although you can save a webpage from your web browser and load the HTML file, Readability Studio also has the convenient ability to directly connect to a webpage. To do this, open the webpage from any web browser and copy the webpage's URL onto the clipboard.

Next, we will create a new project using this URL. There are two ways to accomplish this. The first is to click the New Project from Clipboard... button on the toolbar. This option will take what is currently on the clipboard and use it as the input for a new project. If you have a large block of text on the clipboard, then that will be what is analyzed. However, if you have a local file path or URL on the clipboard, then the project will load the file or webpage.

The other option is to click the New Project... button, select Read text from a file or webpage, and enter the URL in the file path field, as illustrated below.

After specifying the URL, choose to analyze the page by document type and then select Technical document. Most webpages would fall under the category of either general or technical document. Because this page discusses the subject of the Linux operating system, we will select technical.

Next, accept the rest of the defaults on the New Project wizard. Once the project is finished loading, click on the View Document icon to display the highlighted content of the webpage.

Note how some text is highlighted gray. These are incomplete sentences, and because Readability Studio, by default, ignores incomplete sentences, nothing highlighted gray is factored into the analysis. That is, nothing highlighted gray here counts towards the overall sentence, paragraph, and word totals and are also not included in any readability test calculation. Aside from non-narrative forms, it is normally recommended to ignore incomplete sentences. This is especially true for webpages, because often HTML tables and lists are used on websites to display menus--items that are not normally part of the page's narrative.

If you wish to include incomplete sentences in the analysis, select Project Settings... from the File menu to display the Project Properties dialog. Next, click the Document Analysis icon, and select the Include in analysis and treat as full sentences option.

Note that this option is also available on the Options dialog (from the Tools menu) if you wish to change this behavior for all future projects.

One last item to discuss about loading a webpage is whether you want to be able to reload it later on. By default, the original text from the website will be saved into your project, and whenever you open this project, the results will be based off the original page. If you prefer to be able to reload the page (perhaps because it is a dynamic page that periodically changes), then select Project Properties... from the File menu to display the Project Settings dialog. Then click the Document Analysis icon, and select the Reload the source document when opening project option.

By selecting this option, you will reload the page's content when you reopen the project (or at anytime by pressing F5).

Note that webpages requiring authentication (i.e., requiring a user name and password) are not currently supported. To work around this, log into the webpage from any browser, save the page as a local HTML file, and then load it in a new project.
