Creating a Custom Familiar Word Test

Along with offering standard familiar-word tests, such as New Dale Chall and Revised Spache, Readability Studio also includes the ability to create your own familiar-word test. Many customizable features are offered for new tests, such as being able to select our own familiar-word list and which formula to use.

To create a custom familiar-word test, go to the Readability menu and select Add Test... from the Custom Tests submenu. At this point, the Add Custom Word Test dialog will be displayed:

Enter a name for this test and then enter the path to the familiar-word file. This file must be a text file and each word must be separated by a space, tab, or new line. An example file would look like this:

apple orange pear

or like this:

apple
orange
pear

After selecting the word file, click the View familiar word list button to view the words imported from the file.

The next step is to decide which familiar-word test formula to use. Please refer to New Dale Chall (the default) and Revised Spache to help decide which test you prefer.

Next, you can specify which type of stemming (if any at all) to use. Stemming compares words based on their roots and is useful for finding words that appear in different forms. This comparison is performed when words from a document are being matched against your familiar-word list. For example, let us say that you have the word "open" in your list and have English stemming selected. When you analyze a document, words such as "opened" and "opening" will be considered familiar, even if they are not explicitly in your word list.

If you are not using stemming, then it is recommended that you enter each form of your words (e.g., possessive, plural) in your familiar-word list. For example, you would need to have "open", "opens", "opening", etc. explicitly in your list if you want each form of "open" to be considered familiar.<

Standard familiar-word tests normally consider numbers and proper nouns as familiar. This is the default behavior for a custom test as well, but these can be toggled with the options shown below.

Along with your own word list, you can also use the Dale-Chall and Spache word lists in conjunction with it. This enables you to make your word list an appendage to the Dale-Chall word list, therefore creating your own extended Dale-Chall test. This behavior can be toggled with the options shown below.

Finally, you can specify how this test interacts with the New Project Wizard by associating it with document types and industries. Click the Project Wizard Settings button to display these options. Then select all of the documents and industries that this test will relate to.

For example, select the Children's literature and Children's healthcare check boxes (as shown below) and click OK:

Now create a new project and select Children's healthcare as the industry that your document relates. Once the program loads the document, your custom test will be automatically included in the results.
